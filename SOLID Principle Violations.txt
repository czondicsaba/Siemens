SOLID Principle Violations

In ItemController.cs, the single responsibility principle is violated at line 18 and 43. The GetAll and GetById methods violate it because they make the controller do too many things: log, calculate, format, validate. The controller should only receive HTTP requests, give work to services and return HTTP responses. I can solve this violation by creating new classes for the tasks the controller handled until this moment, so that it can focus on its only responsibility. I created service classes, one for each task. I wanted to make classes dependent on interfaces, so I created interfaces for the methods in the Interfaces folder, while the classes were added in the Services folder. With this solution, I make sure that the Dependency Inversion principle is also not violated. Am restructurat codul original al controller-ului.
